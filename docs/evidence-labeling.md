# Evidence Labeling Guide — EpistemicFactKG

Consolidated reference for per-evidence fields, labeling rules, and stance design.
Original decisions: [ADR-010](adr/010-per-evidence-epistemic-modeling.md), [ADR-011](adr/011-evidence-labeling-rules.md), [ADR-028](adr/028-generator-direct-v3-output.md).
See also: [labeling_guide.md](labeling_guide.md) for annotator-facing detail.

---

## Per-Evidence Fields (schema v3.0)

Each evidence item in `record["evidence"]` must have:

| Field | Type | Description |
|-------|------|-------------|
| `evidence_id` | str | Unique ID within claim |
| `text` | str | Evidence text (answer-only for AVeriTeC Q+A) |
| `modality` | enum | `sensor`, `web_text`, `pdf`, `structured_db`, `unknown` |
| `stance` | enum | `supports`, `refutes`, `not_enough_evidence`, `conflicting_evidence` |
| `evidence_types` | list[str] | Pramana types (multi-label) |
| `source_id` | str | Registry key for ST lookup |
| `inference_strength` | float | IS rubric value [0, 1] |
| `source_url` | str\|null | Original URL (AVeriTeC) |
| `triples` | list\|null | RDF triples (AI2THOR only) |

---

## Stance Labels

Four valid values (ADR-028 removed `absent` as a fifth value):

| Stance | Meaning |
|--------|---------|
| `supports` | Evidence confirms the claim |
| `refutes` | Evidence contradicts the claim |
| `not_enough_evidence` | Insufficient or off-topic evidence |
| `conflicting_evidence` | Internal contradiction within one evidence item |

**`absent` was abolished (ADR-028):** Sensor-confirmed absence (`non_apprehension`) is
modeled as `stance: "supports"` + `evidence_types: ["non_apprehension"]`. The epistemic
type carries the absence semantics; the stance direction is still `supports`.

**`conflicting_evidence` is excluded from GNN training (ADR-007):** Records whose only
evidence has `conflicting_evidence` stance are filtered at graph-build time.

## Labeling Rules by Source

### AI2THOR (simulator)

All evidence is generated by the simulator — `assignment_method: "simulator"`.

| Reasoning Strategy | evidence_types | stance | IS |
|---|---|---|---|
| `direct_observation` | `["perception"]` | depends on claim | 1.0 |
| `absence_detection` | `["perception", "non_apprehension"]` | `supports` (absence claim) | 1.0 |
| `spatial_reasoning` | `["perception", "comparison_analogy"]` | depends | 1.0 |
| `action_testing` | `["perception", "inference"]` | depends | 1.0 |

`source_id = "sensor_perception"`, `modality = "sensor"`.

### AVeriTeC

`assignment_method: "heuristic"`. IS derived from `answer_type` ([ADR-019](adr/019-averitec-is-rubric.md)):
- Extractive answer → IS = 0.8
- Abstractive answer → IS = 0.6
- Boolean answer → IS = 0.8

NLI preprocessing (ADR-025): strip "Question: … Answer:" prefix before NLI scoring; only the
answer portion is sent to the NLI cross-encoder.

### Synthetic

`assignment_method: "llm_generated"` or `"rule_based"`. IS values are template-assigned then
jittered with `N(0, σ=0.05)` clamped to [0.1, 1.0] ([ADR-017](adr/017-is-jitter-synthetic.md)).

## Evidence Types — Multi-Label Rules

An evidence item may have multiple Pramana types. The combined EW uses product-of-complements.
`postulation_derivation` is excluded from GNN training but valid in the schema.

| Evidence type combination | Typical scenario |
|---|---|
| `["perception"]` | Direct sensor observation |
| `["perception", "non_apprehension"]` | Sensor-confirmed absence |
| `["testimony"]` | Web text / citation |
| `["testimony", "inference"]` | Multi-hop web synthesis |
| `["perception", "comparison_analogy"]` | Spatial/numeric comparison from simulation |
