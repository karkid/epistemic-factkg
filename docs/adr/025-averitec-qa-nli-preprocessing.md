# ADR-025: AVeriTeC Q+A Evidence Pre-processing for NLI Scoring

**Status:** Accepted  
**Date:** 2026-05-18  
**Builds on:** ADR-024 (NLIHybridHGNN)

---

## Context

AVeriTeC evidence items are stored as natural-language Q+A pairs generated by
fact-checkers during their investigation:

```
"Question: Did the agency officially acknowledge the error?
 Answer: Yes, the agency officially acknowledged the error in their annual report."
```

The NLI cross-encoder (`cross-encoder/nli-deberta-v3-small`) is trained on
natural-language sentence pairs as `(premise, hypothesis)`. When a Q+A string
is used as the premise, the question clause pollutes the inference:

| What NLI receives as premise | NLI behaviour |
|---|---|
| Full Q+A string | NLI treats the question as context; entailment score is biased toward the question's framing |
| Answer-only string | NLI receives a declarative premise; correct entailment/contradiction signal |

**Empirical failure mode** (before fix): AVeriTeC claims where the evidence answer
clearly supported the claim were scored `neutral ≈ 0.99` because the NLI model
could not resolve the stance of a Q+A wrapper against the claim hypothesis.

The AVeriTeC Q+A format is specific to that dataset; AI2THOR and synthetic evidence
are already in declarative form and require no transformation.

---

## Decision

Before passing AVeriTeC evidence to `encode_nli_stance`, strip the
`"Question: … Answer:"` prefix and retain only the answer portion.

**Implementation** (`src/model/data/featurizer.py`):

```python
_QA_ANSWER_RE  = re.compile(r"(?i)^.*?answer\s*:\s*", re.DOTALL)
_MIN_ANSWER_LEN = 15   # below this → answer is uninformative (date, "No", "N/A")

def _extract_nli_text(self, text: str) -> str:
    m = _QA_ANSWER_RE.match(text)
    if m:
        answer = text[m.end():].strip()
        if len(answer) >= _MIN_ANSWER_LEN:
            return answer
    return text   # fallback: non-AVeriTeC or very short answer
```

The 15-character threshold filters out answers such as `"No."`, `"2019."`,
`"Yes, confirmed."` that carry no useful premise for NLI without their question.
For these short answers, the original Q+A string is retained rather than discarding
the evidence — a noisy NLI score is preferable to losing evidence entirely.

The transformation is applied at graph-build time, before NLI inference, so the
graph dataset already contains the corrected NLI probs. No change to graph schema.

---

## Alternatives Considered

**A. Prepend the question to the hypothesis (claim):** Reformulate the NLI pair as
`(answer, "Q: [question] → [claim]")`. Adds question context but makes the hypothesis
unnatural; cross-encoder performance on out-of-domain hypothesis formats is unknown.
Rejected — the answer-only approach is simpler and consistent with how the NLI model
was trained.

**B. Replace Q+A with abstractive summary:** Use an LLM to summarise the Q+A pair
into a single declarative sentence before NLI scoring. Accurate but adds a second
large model to the graph build pipeline. Deferred as future work.

**C. Drop very short answers:** Filter out evidence items whose extracted answer is
below the minimum length instead of falling back to the original text. Rejected —
losing evidence items changes the graph topology and may hurt models that rely on
evidence count. The fallback noisy NLI score is less harmful than a missing node.

---

## Results

AVeriTeC verdict accuracy on the test set (v3-nli):

| Configuration           | AVeriTeC Acc | Overall Acc |
|-------------------------|-------------|-------------|
| Before fix (Q+A as-is)  | ~55%        | ~75%        |
| After fix (answer-only) | **67.4%**   | **79.3%**   |

The improvement (+12pp on AVeriTeC, +4pp overall) is the largest single-fix gain
in the v3-nli development history.

---

## Consequences

- AVeriTeC-sourced evidence with short answers (< 15 chars) retain their original
  Q+A text for NLI scoring. These are expected to produce neutral ≈ 0.99 scores,
  which are informative (the evidence is ambiguous w.r.t. the claim).
- Non-AVeriTeC evidence (AI2THOR, synthetic) is unaffected — the regex only matches
  when the `"Answer:"` keyword is present.
- The NLI cross-encoder is not fine-tuned or changed; only the input preparation
  changes. The frozen DeBERTa model produces better-calibrated probs when given
  declarative premises.
